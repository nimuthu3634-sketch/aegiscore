## Getting Started

### Logging In

Navigate to the AegisCore web interface and log in with your email and password. Contact your administrator if you do not have an account.

After login you are taken to the **Dashboard**, which gives a live overview of the SOC's security posture.

### Changing Your Password

1. Click your name in the top-right corner and select **Profile**.
2. Enter your current password and a new password (minimum 8 characters, must include uppercase, lowercase, number, and symbol).
3. Click **Save**.

## Dashboard

The dashboard provides a real-time summary of security activity.

| Panel | Description |
|-------|-------------|
| Open Alerts | Total unresolved alerts across all sources |
| Critical / High Alerts | Count of the highest-severity open alerts |
| Open Incidents | Active security cases requiring investigation |
| Logs Today | Number of log entries ingested today |
| Alert Trend (7 days) | Severity breakdown over the past week |
| Top At-Risk Assets | Assets with the highest current risk scores |
| Integration Health | Live status of Wazuh, Suricata, Nmap, and Hydra connectors |
| Recent Activity | Latest alert and incident events |

Alerts are pushed to the dashboard in real time via WebSocket — no page refresh needed.

## Alerts

### Viewing Alerts

Go to **Alerts** in the sidebar. Use the filter bar to narrow results by:
- **Source** (wazuh, suricata, nmap, hydra)
- **Severity** (critical, high, medium, low)
- **Status** (open, triaged, investigating, resolved, suppressed)
- **Assignee**
- **Incident**

Click any alert row to open its detail page.

### Alert Detail Page

The detail page shows:
- **Overview** — severity, status, source, event type, timestamps
- **Risk Score** — a 0–100 score generated by the ML model, with an explanation summary listing the factors that influenced the score
- **Response Recommendations** — suggested analyst actions in priority order
- **Comments** — threaded discussion visible to all analysts
- **Linked Incidents** — cases this alert has been attached to

### Updating an Alert

Analysts and Admins can:
- Change **Status** (open → triaged → investigating → resolved / suppressed)
- Change **Severity**
- **Assign** the alert to an analyst
- Add **Tags**
- Edit the **AI explanation summary** with manual context

Click **Save** after making changes.

### Creating an Incident from an Alert

On the alert detail page, click **Create Incident**. Fill in the title, priority, and optional description. The alert is automatically linked to the new incident.

### Executing a Response Action

On the alert detail page, click **Respond**. Choose one of the available automated actions:

| Action | Effect |
|--------|--------|
| Block IP | Records a block recommendation for the source IP |
| Isolate Asset | Flags the associated asset for isolation |
| Disable User | Flags the associated user account for deactivation |
| Contain Alert | Marks the alert as contained and updates its status |

All response actions are logged to the audit trail.

## Incidents

### Viewing Incidents

Go to **Incidents** in the sidebar. Filter by status (open, contained, monitoring, resolved) or priority (P1–P4).

### Incident Detail Page

Each incident has:
- **Overview** — reference number, priority, status, assignee
- **Timeline** — chronological record of notes, status changes, and linked alerts
- **Linked Alerts** — all alerts attached to this case

### Managing an Incident

Admins and Analysts can:
- Change **Status** and **Priority**
- **Assign** to an analyst
- Add **Timeline Notes**
- **Link** additional alerts to the incident
- **Unlink** alerts that are no longer relevant

### Closing an Incident

Set the status to **Resolved** and add resolution notes in the timeline to document the outcome.

## Integrations

### Overview

Go to **Integrations** to see all configured data sources.

| Integration | Type | How Data Enters |
|-------------|------|-----------------|
| Wazuh | SIEM / Endpoint | Manual sync or agent push |
| Suricata | Network IDS | Manual sync |
| Nmap | Port Scanner (Lab) | File import only |
| Hydra | Brute-Force Sim (Lab) | File import only |

### Configuring Wazuh or Suricata

1. Click the integration card.
2. Click **Configure**.
3. Enter the **Endpoint URL** of your Wazuh or Suricata API.
4. Select the **Auth Type** (none, bearer token, or basic auth) and provide credentials.
5. Set **Lookback Minutes** (how far back to fetch events on each sync).
6. Toggle **TLS Verification** on for production environments.
7. Click **Save**.

After configuring, click **Sync Now** to pull the latest events immediately. The integration health indicator updates automatically.

### Importing Nmap or Hydra Results (Lab)

Nmap and Hydra are lab-only connectors — AegisCore never executes scans or attacks. You import the output files produced by your own lab machines.

1. Click the Nmap or Hydra card.
2. Click **Import File**.
3. Upload the result file:
   - Nmap: `.xml` (nmap XML output) or `.json`
   - Hydra: `.txt` (raw Hydra console output) or `.json`
4. AegisCore parses the file, creates alerts for findings, and records a run entry.

### Viewing Run History

Click **Run History** on any integration card to see a log of all past syncs and imports, including record counts and any errors.

## Logs

Go to **Logs** to search raw and parsed log entries across all integrations.

Filter by:
- **Source** (wazuh, suricata, nmap, hydra)
- **Category** (endpoint, network, exposure, credential-exposure, etc.)
- **Asset**
- **Time Range**

Each row shows the log level, message, and event timestamp. Click a row to expand the raw and parsed payload.

## AI Risk Scoring

Go to **AI** in the sidebar to view the active ML risk model.

### What the Risk Score Means

Each alert receives a score from **0 to 100**:

| Band | Score Range | Meaning |
|------|-------------|---------|
| Critical | 75–100 | Immediate attention required |
| High | 50–74 | Investigate promptly |
| Medium | 25–49 | Review during normal workflow |
| Low | 0–24 | Monitor, low immediate impact |

### Explainability Factors

The score is explained by up to 15 factors including:
- Alert severity and source type
- How frequently similar alerts have fired (1h / 24h / 7d windows)
- Asset criticality and number of open alerts on the asset
- Whether the event occurred outside business hours
- Correlated alerts from multiple sources in the last 24 hours
- Whether the data was imported from a lab tool (Nmap/Hydra)

### Retraining the Model

Admins can click **Retrain Model** to trigger a new training run on all historical alerts. The job runs in the background and the new model becomes active automatically once training completes. Previous model versions are preserved in the model history table.

### Recalculating Scores

Click **Recalculate All Scores** to re-score every open alert with the currently active model without retraining.

## User Management (Admin)

Admins can manage user accounts under **Admin** in the sidebar.

### Creating a User

1. Click **New User**.
2. Enter the user's full name, email address, and initial password.
3. Select a **Role** (Admin, Analyst, or Viewer).
4. Click **Create**.

The user can log in immediately and should change their password on first login via the Profile page.

### Editing a User

Click a user row to edit their name, role, or reset their password.

### Deactivating a User

Click the user row and toggle **Active** off. Deactivated users cannot log in but their history is preserved.

### Audit Log

The **Audit** tab shows a full log of all system actions (alert updates, incident changes, response actions, user changes, integration syncs) with actor, timestamp, and IP address.

## Troubleshooting

**Integration shows Offline / Degraded**
- Check the endpoint URL is reachable from the AegisCore server.
- Verify credentials and TLS settings.
- Click **Sync Now** and check the error message in Run History.

**No alerts appearing after import**
- Confirm the file format matches the integration (Nmap: XML or JSON, Hydra: TXT or JSON).
- Check Run History for parse errors.
- Ensure the file contains at least one result record.

**Risk scores not updating**
- Go to **AI** and click **Recalculate All Scores**.
- If scores still seem stale, trigger a **Retrain Model** run.

**Cannot log in**
- Check your email and password.
- Contact an Admin — your account may be deactivated.
- Auth is rate-limited to 5 attempts per minute; wait 60 seconds and try again.
